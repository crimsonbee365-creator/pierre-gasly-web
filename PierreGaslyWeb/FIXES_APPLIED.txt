VERSION 3.0 SPECIFIC CHANGES:
=============================

1. FETCHONE METHOD - COMPLETE REWRITE
   -----------------------------------
   a) Added pattern matching for SELECT specific_column queries
      - BEFORE: Only handled SELECT * FROM table WHERE column = ?
      - AFTER: Also handles SELECT column FROM table WHERE column = ?
      - This fixes email existence checks
   
   Example:
   ```sql
   SELECT user_id FROM users WHERE email = ?
   ```
   Now properly returns the user_id if email exists, or null if not.

2. UPDATE METHOD - BOOLEAN CONVERSION
   ------------------------------------
   a) Added automatic boolean conversion for is_active fields
      - When updating is_active = 0, converts to boolean false
      - When updating is_active = 1, converts to boolean true
      - PostgreSQL requires boolean type for boolean columns
   
   b) Applies to all fields starting with "is_"
      - is_active, is_verified, is_deleted, etc.
      - Numeric values automatically converted to boolean
   
   Example:
   ```sql
   UPDATE products SET is_active = 0 WHERE product_id = 5
   ```
   Data sent to Supabase: {"is_active": false}

3. INSERT METHOD - COMPLETE REWRITE  
   ----------------------------------
   a) Completely rewrote INSERT query parsing
      - Better handling of parameter mapping
      - Proper value tokenization
      - Support for literal values and functions
   
   b) Handles all INSERT patterns:
      - All placeholders: VALUES (?, ?, ?)
      - Mixed with literals: VALUES (?, 'active', ?)
      - With functions: VALUES (?, NOW(), ?)
   
   c) Boolean field conversion on insert
      - Automatically converts numeric values for is_* fields
      - Ensures proper boolean type in PostgreSQL
   
   Example:
   ```sql
   INSERT INTO products (..., is_active, ...) VALUES (..., 1, ...)
   ```
   Data sent: {..., "is_active": true, ...}

4. SELECT METHOD - IMPROVED BOOLEAN HANDLING
   -----------------------------------------
   a) Added special handling for is_active queries
      - Integer 0 automatically converted to "eq.false"
      - Integer 1 automatically converted to "eq.true"
      - Ensures proper filtering of archived vs active records
   
   b) Applied to all fields starting with "is_"
   
   Example:
   ```php
   $db->select('products', ['is_active' => 1])
   ```
   Generates URL: /products?select=*&is_active=eq.true

5. DEBUG MODE ADDED
   -----------------
   a) Added comprehensive debugging system
      - Enable with ?debug_db=1 URL parameter
      - Logs all queries, parameters, and results
      - Helps identify issues quickly
   
   b) Logs include:
      - Query text and parameters
      - Data sent to Supabase
      - Results returned
      - Errors encountered
   
   Usage:
   http://yourdomain.com/products.php?debug_db=1

6. ERROR LOGGING ENHANCED
   -----------------------
   a) Improved error messages
      - Shows HTTP status code
      - Shows Supabase error message
      - Shows data that was sent
      - Helps debug issues faster
   
   b) JSON decode error handling
      - Detects malformed JSON responses
